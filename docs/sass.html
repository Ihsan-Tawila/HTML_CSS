SASS

Intro

Når du designer store og komplekse nettsteder, blir stilarkene lange, mer komplekse og vanskelige å vedlikeholde. Derfor var det nødvendig å lage et utvidelsesspråk til CSS for å gjøre css-filen mer strukturert.
I denne artikkelen vil vi lære om sass-språket, som har tilleggsfunksjoner som ikke finnes i css, for eksempel variabler, hekk, mixins, arv og andre smarte godbiter som gjør skriving av CSS mer letere.

Hva er SASS?

Når du designer store og komplekse nettsteder, blir stilarkene lange, mer komplekse og vanskelige å vedlikeholde. Derfor var det nødvendig å lage et utvidelsesspråk til CSS for å gjøre css-filen mer strukturert. I denne artikkelen vil vi lære om sass-språket, som har tilleggsfunksjoner som ikke finnes i css, for eksempel variabler, hekk, mixins, arv og andre smarte godbiter som gjør skriving av CSS mer letere.

Nesting av selectorer

I CSS-filen må du bruke stilregelsett på hvert element separat men i sass-filen kan du legge flere stilregelsett inni hverandre for å kun gjelde for elementer som er omsluttet av den. For eksempel når vi vil lage en liste:

************* SCSS *************
nav {
  ul {
    margin: 0;
    padding: 0;
    list-style: none;
  }

  li { display: inline-block; }

  a {
    display: block;
    padding: 6px 12px;
    text-decoration: none;
  }
}
************ css ****************
nav ul {
  margin: 0;
  padding: 0;
  list-style: none;
}
nav li {
  display: inline-block;
}
nav a {
  display: block;
  padding: 6px 12px;
  text-decoration: none;
}

Variabler

I Sass kan vi ha variabler og det er et kodenavn som vi kan gjenbruke videre i Sass-filen. Det løster et evigvarende problem for designere. F.eks. å huske og gjenbruke fargekoder! Vi kan bruke variabler til mye mer enn fargekoder. Vi kan si at variabler er som en måte å lagre informasjon vi vil bruke på nytt gjennom stilarket våre.

************* SCSS *************
$font-stack:Helvetica, sans-serif;
$primary-color: #333;

body {
  font: 100% $font-stack;
  color: $primary-color;
}
}
************ css ****************
body {
  font:100% Helvetica, sans-serif;
  color: #333;
}

Mixins

Noen stilregelsetter har mange lange koder som vi må skrive hver gang vi vil bruke dem. Men i sass kan vi skrive den en gang i form av en funksjon og gi den et spesifikt navn og bruke den igjen når du trenger ved å skrive navnet til funksjonen.

************* SCSS *************
@mixin transform($property) {
  -webkit-transform: $property;
  -ms-transform: $property;
  transform: $property;
}
.box{@include transform(rotate(30deg));}
************ css ****************
.box {
  -webkit-transform: rotate(30deg);
  -ms-transform: rotate(30deg);
  transform: rotate(30deg);
}

Extend

Dette er en av de mest nyttige funksjonene i Sass. Ved å bruke @extend kan du dele et sett med CSS-egenskaper fra en velger til en annen. I vårt eksempel skal vi lage en enkel serie med meldinger for feil, advarsler og suksesser ved hjelp av en annen funksjon som går hånd i hånd med utvidede plassholderklasser. En plassholderklasse er en spesiell type klasse som bare skrives ut når den utvides, og som kan bidra til å holde den kompilerte CSS-en ren og ren.

************* SCSS *************
/* This CSS will print because %message-shared is extended. */
%message-shared {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

// This CSS won't print because %equal-heights is never extended.
%equal-heights {
  display: flex;
  flex-wrap: wrap;
}

.message {
  @extend %message-shared;
}

.success {
  @extend %message-shared;
  border-color: green;
}

.error {
  @extend %message-shared;
  border-color: red;
}

.warning {
  @extend %message-shared;
  border-color: yellow;
}
************ css ****************
/* This CSS will print because %message-shared is extended. */
.message, .success, .error, .warning {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.success {
  border-color: green;
}

.error {
  border-color: red;
}

.warning {
  border-color: yellow;
}
